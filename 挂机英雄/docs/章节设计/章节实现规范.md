# 章节实现规范

> 本文档记录**新增一个完整章节**的具体实现步骤，以第五章「深海渊域」为实例。
> 后续新增第六章及之后的章节时，按本文档逐步操作即可。

---

## 一、实现总览

### 1.1 需要修改/新增的文件清单

| 文件 | 操作 | 内容 |
|------|------|------|
| `scripts/StageConfig.lua` | 追加 | 怪物模板 + 章节关卡定义 |
| `scripts/Config.lua` | 追加 | 2 套新装备套装 + 套装图标路径 |
| `scripts/GameState.lua` | 追加（仅新机制时） | 新 debuff 状态变量 |
| `scripts/state/Combat.lua` | 追加（仅新机制时） | debuff 施加/衰减函数 |
| `scripts/state/StatCalc.lua` | **无需手动改动** | 属性计算已走 SM.Apply，新 debuff 只需 SM.Register |
| `scripts/state/StatModifiers.lua` | 追加（仅新机制时） | SM.Register 注册新 debuff 对属性的影响 |
| `scripts/battle/Spawner.lua` | 追加（仅新机制时） | 新机制字段透传 |
| `scripts/battle/EnemySystem.lua` | 追加（仅新机制时） | 新机制运行时逻辑 |
| `assets/Textures/mobs/` | 新增 | 怪物/Boss 贴图 |
| `assets/Textures/` | 新增 | 章节选关背景图 + 战斗背景图 |
| `assets/` | 新增 | 套装图标 + Boss 技能特效图 |
| `scripts/BattleView.lua` | 追加 | `CHAPTER_BG` 表注册战斗背景 |
| `scripts/view/DrawEffects.lua` | 追加（仅新元素时） | `BOSS_SKILL_IMAGES` 表注册技能图 |

### 1.2 实现顺序（必须遵守）

```
1. StageConfig.lua — 怪物模板 (MONSTERS 表)
2. StageConfig.lua — 章节关卡 (CHAPTERS 表)
3. Config.lua — 装备套装 (EQUIP_SETS 表 + SET_ICON_PATHS)
4. 新机制代码（如有）:
   a. GameState.lua — 状态变量
   b. Combat.lua (或 BuffRuntime.lua) — 施加/衰减函数 + ResetHP 清理
   c. StatModifiers.lua — SM.Register 注册属性影响（替代旧的 StatCalc 硬编码）
   d. Spawner.lua — 字段透传
   e. EnemySystem.lua — 运行时逻辑
5. 生成图片资源:
   a. 怪物/Boss 贴图 (8+2 张)
   b. 套装图标 (2 张)
   c. 选关背景 (1 张)
   d. 战斗背景 — 2D 俯视角 (1 张)
   e. Boss 技能特效图 (按需，新元素时)
   f. 套装触发特效图 (按需，有主动触发效果的套装)
6. 图片代码注册:
   a. BattleView.lua — CHAPTER_BG 表追加战斗背景
   b. DrawEffects.lua — BOSS_SKILL_IMAGES 表追加技能图（仅新元素）
   c. DrawEffects.lua / BuffManager.lua — 套装触发特效图注册（见 §6.3）
7. 构建 & 验证
```

## 三、StageConfig.lua — 章节关卡定义

### 3.1 插入位置

在 `StageConfig.CHAPTERS` 数组末尾追加新章节。

### 3.2 关卡结构

```lua
{
    id = N,
    name = "章节名称",
    desc = "一句话描述",
    lore = "80-150 字背景故事",
    stages = {
        -- 固定 10 关，每关结构：
        {
            name = "关卡名称",
            isBoss = nil,          -- 仅 5、10 关为 true
            waves = {
                { monsters = {
                    { id = "怪物ID", count = 数量 },
                }},
                -- 2-3 波
            },
            scaleMul = 数值,       -- 难度系数
            reward = { gold = 数值, guaranteeEquipQuality = N },
        },
    },
},
```

### 3.3 scaleMul 计算规则

| 计算步骤 | 公式 | 说明 |
|----------|------|------|
| 起始值 | 上一章末关 scaleMul × 1.3~1.5 | 确保过渡关"上一章毕业装可轻松通过" |
| 章内终止值 | 起始值 × 5~6 | 各章章内倍率约 4.8~5.8 |
| 过渡关增幅 | ×1.07~1.10 | 低增幅，缓冲适应区 |
| 普通关增幅 | ×1.13~1.25 | 中等压力 |
| 装备墙增幅 | ×1.20~1.30 | 高增幅，迫使换装 |
| Boss 关增幅 | ×1.25~1.35 | 最高增幅，套装检定 |

> **注意**：增幅不是均匀 ×1.2，而是"缓-急"节奏——前期过渡关低增幅，装备墙/Boss 关高增幅。

**各章参考数据**：

| 章节 | 首关 | 末关 | 章内倍率 |
|------|------|------|---------|
| 1 | 1.4 | 7.8 | ×5.6 |
| 2 | 12 | 60 | ×5.0 |
| 3 | 88 | 422 | ×4.8 |
| 4 | 600 | 3,460 | ×5.8 |
| 5 | 4,600 | 24,600 | ×5.3 |

第五章实例：3,460 × 1.33 ≈ 4,600（起始）→ 24,600（末关，×5.3）

### 3.4 Boss 关结构（第 5 关、第 10 关）

## 五、新战斗机制实现（仅当章节引入新机制时需要）

> 如果新章节只复用已有机制（如 slowOnHit、lifesteal 等），跳过本节。

### 5.1 实现路径总览

新机制需要在 **5 个文件** 中按固定模式添加代码：

```
GameState.lua        → 声明状态变量
BuffRuntime.lua      → 施加函数 + 衰减逻辑 + ResetHP 清理
StatModifiers.lua    → SM.Register 注册属性影响（如影响 ATK/DEF/CritRate）
Spawner.lua          → 字段透传
EnemySystem.lua      → 运行时触发逻辑
```

> **注意**: StatCalc.lua 的 Get* 函数已全部走 `SM.Apply()`，
> 新 debuff 只需在 BuffRuntime.lua 末尾添加一行 `SM.Register({...})`，
> StatCalc 不再需要手动修改。

### 5.2 步骤 A: GameState.lua — 声明状态变量

**位置**: 在 `GameState.playerSlowTimer` 之后追加。

**模式**:

```lua
-- ★ 新机制名称
GameState.新变量1 = 初始值    -- 注释说明
GameState.新变量2 = 初始值    -- 注释说明
```

**第五章实例**:

```lua
-- 腐蚀 debuff（第五章深渊巨蟹）
GameState.corrosionStacks = 0        -- 腐蚀层数 (降低DEF)
GameState.corrosionDefReduce = 0     -- 每层DEF降低比率
GameState.corrosionTimer = 0         -- 腐蚀debuff剩余时间
GameState.corrosionMaxStacks = 0     -- 腐蚀最大层数

-- 墨汁致盲 debuff（第五章墨渊章鱼）
GameState.inkBlindRate = 0           -- 墨汁致盲ATK降低比率
GameState.inkBlindTimer = 0          -- 墨汁debuff剩余时间
```

### 5.3 步骤 B: Combat.lua — 施加函数 & 衰减 & 重置

#### B1. 施加函数

**位置**: 在已有 `ApplyAntiHealDebuff` 函数之后追加。

**模式（叠加型 debuff）**:

```lua
GameState.Apply新Debuff = function(参数...)
    if GameState.playerDead then return end
    -- 设置参数
    GameState.xxx = 参数值
    GameState.xxxTimer = duration  -- 刷新持续时间
    -- 叠加逻辑
    if GameState.xxxStacks < maxStacks then
        GameState.xxxStacks = GameState.xxxStacks + 1
    end
end
```

**模式（覆盖型 debuff，取最强值）**:

```lua
GameState.Apply新Debuff = function(参数...)
    if GameState.playerDead then return end
    -- ★ 韧性抗性衰减（所有 debuff 必须经过此计算）
    local resist = GameState.GetDebuffResist()
    local durFactor = Config.TENACITY.durFactor  -- 0.5
    local actual参数 = 原始参数 * (1 - resist)
    local actualDur = duration * (1 - resist * durFactor)
    if actual参数 < 0.01 then return end  -- 抗性过高，完全抵抗
    if actual参数 > GameState.当前值 or GameState.xxxTimer <= 0 then
        GameState.当前值 = actual参数
        GameState.xxxTimer = actualDur
    end
end
```

> **韧性抗性规则**：所有新 debuff 的施加函数必须经过 `GetDebuffResist()` 衰减。
> 效果衰减: `actualEffect = rawEffect × (1 - resist)`
> 持续衰减: `actualDur = rawDur × (1 - resist × 0.5)`
> 详见 `Config.TENACITY` 配置（baseResist=0, perPoint=0.02, max=0.80, durFactor=0.5）。

**第五章实例**:

```lua
--- 施加腐蚀 debuff（叠加制，降低DEF）
GameState.ApplyCorrosionDebuff = function(defReducePct, maxStacks, duration)
    if GameState.playerDead then return end
    local resist = GameState.GetDebuffResist()
    local durFactor = Config.TENACITY.durFactor
    local actualReduce = defReducePct * (1 - resist)
    local actualDur = duration * (1 - resist * durFactor)
    if actualReduce < 0.01 then return end
    GameState.corrosionDefReduce = actualReduce
    GameState.corrosionMaxStacks = maxStacks
    GameState.corrosionTimer = actualDur
    if GameState.corrosionStacks < maxStacks then
        GameState.corrosionStacks = GameState.corrosionStacks + 1
    end
end

--- 施加墨汁致盲 debuff（覆盖制，降低ATK）
GameState.ApplyInkBlindDebuff = function(atkReducePct, duration)
    if GameState.playerDead then return end
    local resist = GameState.GetDebuffResist()
    local durFactor = Config.TENACITY.durFactor
    local actualReduce = atkReducePct * (1 - resist)
    local actualDur = duration * (1 - resist * durFactor)
    if actualReduce < 0.01 then return end
    if actualReduce > GameState.inkBlindRate or GameState.inkBlindTimer <= 0 then
        GameState.inkBlindRate = actualReduce
        GameState.inkBlindTimer = actualDur
    end
end
```

#### B2. 衰减逻辑

**模式**:

```lua
    if GameState.xxxTimer <= 0 then
        -- 清除所有相关状态
        GameState.xxx = 0
        GameState.xxxTimer = 0

end
```


### 5.4 步骤 C: StatModifiers.lua — SM.Register 注册属性影响

**仅当新 debuff 影响玩家属性（ATK/DEF/速度/暴击等）时需要**。

> **P0 变更说明**: StatCalc.lua 的 `GetTotalAtk()` / `GetTotalAtkSpeed()` / `GetCritRate()` 已全部走 `SM.Apply()`，
> 新 debuff 只需在 `BuffRuntime.Install()` 末尾添加 `SM.Register({...})`，StatCalc **不再需要手动修改**。

**位置**: 在 `scripts/state/BuffRuntime.lua` 的 `M.Install(GS)` 函数末尾追加。

**模式**:

```lua
-- ★ 新 debuff: 描述
SM.Register({
    id    = "debuff_唯一ID",       -- 全局唯一标识
    stat  = "目标属性",             -- "atk" / "def" / "atkSpeed" / "crit"
    type  = "修饰器类型",           -- "pctReduce"(百分比削减) / "flatSub"(固定减少) / "pctPool"(百分比池) 等
    valueFn = function()
        return 计算值                -- 返回当前生效的数值
    end,
    conditionFn = function()
        return (GameState.xxxTimer or 0) > 0   -- 仅当 debuff 计时器激活时生效
    end,
})
```

**5 种修饰器类型**:

| type | 含义 | 公式中的位置 |
|------|------|------------|
| `pctPool` | 百分比池（加法叠加） | `× (1 + Σ pctPool)` |
| `pctMul` | 百分比独立乘算 | `× Π(1 + pctMul_i)` |
| `pctReduce` | 百分比削减（DR叠加） | `× Π(1 - pctReduce_j)` |
| `flatAdd` | 固定加值 | `(base + flatAdd ...)` |
| `flatSub` | 固定减值 | `(base ... - flatSub)` |

最终公式: `result = (base + flatAdd - flatSub) × (1 + Σ pctPool) × Π(1 + pctMul_i) × Π(1 - pctReduce_j)`

**第五章实例**:

墨汁致盲影响 ATK（pctReduce 类型）:

```lua
SM.Register({
    id    = "debuff_inkBlind",
    stat  = "atk",
    type  = "pctReduce",
    valueFn = function()
        return GameState.inkBlindRate or 0
    end,
    conditionFn = function()
        return (GameState.inkBlindTimer or 0) > 0
    end,
})
```

沙暴影响暴击（flatSub 类型）:

```lua
SM.Register({
    id    = "debuff_sandStorm",
    stat  = "crit",
    type  = "flatSub",
    valueFn = function()
        return GameState.sandStormCritReduce or 0
    end,
    conditionFn = function()
        return (GameState.sandStormTimer or 0) > 0
    end,
})
```

> **注意**: 腐蚀（corrosion）影响 DEF，但 DEF 的计算函数 `GetDEFReduction` 目前尚未迁移到 SM.Apply，
> 仍需在 StatCalc 中手动处理。后续 P1 可将 DEF 也纳入 SM 管理。

#### E2. 死亡触发型机制

**位置**: 在 `OnEnemyDeath` 函数开头（在 deathExplode 检查之前）追加。

**第五章实例 — 分裂 (splitOnDeath)**:

```lua
function EnemySystem.OnEnemyDeath(bs, enemy)
    -- ★ 死亡分裂: 产生子怪物
    if enemy.splitOnDeath then
        local splitCfg = enemy.splitOnDeath
        local childTemplate = StageConfig.MONSTERS[splitCfg.childId]
        if childTemplate then
            local gs = GameState.stage
            local stageCfg = StageConfig.GetStage(gs.chapter, gs.stage)
            local scaleMul = stageCfg and stageCfg.scaleMul or 1.0
            local count = splitCfg.count or 2

            for i = 1, count do
                -- 围绕死亡位置散布
                local angle = (i / count) * math.pi * 2
                local offsetX = math.cos(angle) * 25
                local offsetY = math.sin(angle) * 25
                local sx = math.max(30, math.min(bs.areaW - 30, enemy.x + offsetX))
                local sy = math.max(30, math.min(bs.areaH - 30, enemy.y + offsetY))

                -- 按 scaleMul 缩放属性
                local cHp = math.floor(childTemplate.hp * scaleMul)
                local cAtk = math.floor(childTemplate.atk * scaleMul)

                -- 创建完整的子怪物实例（必须包含所有字段！）
                table.insert(bs.enemies, {
                    x = sx, y = sy,
                    hp = cHp, maxHp = cHp, atk = cAtk,
                    speed = childTemplate.speed,
                    radius = childTemplate.radius or 11,
                    expDrop = math.floor(childTemplate.expDrop * scaleMul),
                    goldMin = math.floor(childTemplate.goldDrop[1] * math.sqrt(scaleMul)),
                    goldMax = math.floor(childTemplate.goldDrop[2] * math.sqrt(scaleMul)),
                    color = { childTemplate.color[1], childTemplate.color[2], childTemplate.color[3] },
                    image = childTemplate.image,
                    isBoss = false, dead = false,
                    def = childTemplate.def or 0,
                    atkTimer = 0,
                    atkCd = childTemplate.atkInterval or 2.0,
                    atkRange = childTemplate.atkRange or 35,
                    name = childTemplate.name,
                    knockbackVx = 0, knockbackVy = 0,
                    element = childTemplate.element or "physical",
                    antiHeal = childTemplate.antiHeal or false,
                    slowOnHit = childTemplate.slowOnHit or 0,
                    slowDuration = childTemplate.slowDuration or 0,
                    attachedElement = nil, attachedElementTimer = 0,
                    defReduceRate = 0, defReduceTimer = 0,
                    reactionDot = nil,
                    templateId = splitCfg.childId,
                    defPierce = childTemplate.defPierce or 0,
                    packBonus = childTemplate.packBonus or 0,
                    packThreshold = childTemplate.packThreshold or 0,
                    isRanged = childTemplate.isRanged or false,
                    deathExplode = childTemplate.deathExplode,
                    hpRegenPct = childTemplate.hpRegen or 0,
                    hpRegenInterval = childTemplate.hpRegenInterval or 0,
                    hpRegenTimer = 0, enraged = false,
                    resist = childTemplate.resist,
                    lifesteal = childTemplate.lifesteal or 0,
                    healAura = childTemplate.healAura, healAuraTimer = 0,
                    firstStrikeMul = childTemplate.firstStrikeMul,
                    firstStrikeDone = false,
                    igniteDot = nil,
                    splitOnDeath = childTemplate.splitOnDeath,
                    corrosion = childTemplate.corrosion,
                    inkBlind = childTemplate.inkBlind,
                })
            end
            Particles.SpawnReactionText(bs.particles, enemy.x, enemy.y - 10, "分裂!", { 200, 100, 80 })
        end
    end
    -- ... 其他死亡处理 ...
end
```

> **关键注意**: 分裂生成的子怪物必须包含 **所有** enemy 实例字段。遗漏任何字段会导致后续代码 `attempt to index a nil value` 错误。参照 `Spawner.SpawnFromQueue` 的完整字段列表。

生成图片后，除了复制到正确路径，还需要在以下代码位置注册：

| 图片类型 | 注册文件 | 注册位置 | 说明 |
|----------|----------|----------|------|
| 战斗背景 | `scripts/BattleView.lua` | `CHAPTER_BG` 表 | 追加 `[N] = "Textures/battle_bg_chN.png"` |
| Boss 技能图 | `scripts/view/DrawEffects.lua` | `BOSS_SKILL_IMAGES` 表 | 按技能类型+元素追加映射 |
| 套装图标 | `scripts/Config.lua` | `SET_ICON_PATHS` 表 | 追加 `套装ID = "文件名.png"` |
| 套装触发特效图 | `scripts/battle/BuffManager.lua` 或 `scripts/view/DrawEffects.lua` | 特效加载 + 渲染函数 | 见下方说明 |
| 选关背景 | 无需注册 | — | 路径由 `StageSelect.lua` 自动拼接 |
| 怪物贴图 | 无需注册 | — | 路径在 `StageConfig.MONSTERS` 的 `image` 字段中直接填写 |

**套装触发特效图注册详解**：

套装特效图的注册方式取决于效果类型：

| 效果类型 | 渲染方式 | 注册位置 |
|----------|---------|---------|
| 爆炸/AOE 型（类似 deathExplode） | 插入 `bs.bossSkillEffects` 队列 | 在 `BOSS_SKILL_IMAGES` 中新增 type 或复用已有 type |
| 护盾/光环型（持续显示） | 在 `DrawEntities` 中独立渲染 | 在 `DrawEntities.lua` 中加载图片句柄 + 渲染逻辑 |
| 瞬间闪光型（反弹/触发） | 在 `BuffManager` 中插入短时特效 | 在 `BuffManager.lua` 中 `nvgCreateImage` 加载 + 闪光渲染 |

**典型注册代码**（以爆炸型为例）：

```lua
-- 方法 1: 复用 bossSkillEffects 队列（推荐，无需改渲染管线）
-- 在 BuffManager.lua 中触发时：
table.insert(bs.bossSkillEffects, {
    type = "deathExplode",           -- 复用已有的爆炸渲染
    element = "water",               -- 会自动查找对应元素的特效图
    x = targetX, y = targetY,
    radius = 80, life = 0.8, maxLife = 0.8,
})

-- 方法 2: 注册新的特效类型（需要独立视觉风格时）
-- 在 DrawEffects.lua 顶部 BOSS_SKILL_IMAGES 后追加：
local SET_EFFECT_IMAGES = {
    ocean_fury_spread = { var = "setOceanFuryHandle", path = "set_effect_ocean_fury_xxx.png" },
    tidal_bastion_reflect = { var = "setTidalReflectHandle", path = "set_effect_tidal_bastion_xxx.png" },
}
-- 然后在 DrawBossSkillEffects 或新函数 DrawSetEffects 中处理渲染
```

### 7.2 常见构建错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Resource not found: Textures/xxx.png` | 图片未放到正确路径 | 检查 `assets/Textures/` 下的文件名 |
| `attempt to index a nil value` | 新机制字段未透传 | 检查 Spawner.lua 是否添加了字段 |
| `attempt to call a nil value` | 施加函数未定义 | 检查 Combat.lua 中的函数是否正确安装 |
| `attempt to perform arithmetic on nil` | 状态变量未初始化 | 检查 GameState.lua 中的变量声明 |
| 战斗场景无背景/黑屏 | `CHAPTER_BG` 表未注册 | 检查 BattleView.lua 中 CHAPTER_BG 是否有新章节条目 |
| Boss 技能无特效图/显示纯色圆 | `BOSS_SKILL_IMAGES` 表未注册 | 检查 DrawEffects.lua 中对应元素是否已映射 |
