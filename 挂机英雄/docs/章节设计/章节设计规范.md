# 章节设计规范

> **版本**: v3.0 | **更新日期**: 2026-03-10
>
> 本文档定义了新增章节时需要遵循的标准流程和数据格式。
> 按照本规范填写，可以确保新章节与现有系统无缝对接。

---

## 一、章节总览

### 1.1 已有章节参照

| 章节 | 名称     | 主题色调   | 主元素     | Tier 系数 | attrScale | scaleMul 范围   |
|------|----------|------------|------------|-----------|-----------|-----------------|
| 1    | 灰烬荒原 | 灰黄       | 火/毒/物理 | 1         | 1.00      | 1.4 ~ 7.8       |
| 2    | 冰封深渊 | 冰蓝       | 冰/水      | 2         | 1.15      | 12 ~ 60          |
| 3    | 熔岩炼狱 | 熔红       | 火/毒/物理 | 4         | 1.30      | 88 ~ 422         |
| 4    | 幽暗墓域 | 幽紫/暗绿  | 奥术/物理  | 8         | 1.45      | 600 ~ 3,460      |
| 5    | 深海渊域 | 深蓝/荧光  | 水/冰      | 16        | 1.60      | 4,600 ~ 24,600   |
| 6    | 雷鸣荒漠 | 金黄/电紫  | 雷/物理    | 32        | 1.75      | （见 StageConfig）|
| 7    | 瘴毒密林 | 暗绿/毒紫  | 毒/自然    | 64        | 1.90      | （见 StageConfig）|

### 1.2 核心公式

| 参数 | 公式 | 当前值 | 说明 |
|------|------|--------|------|
| Tier 系数 | `2 ^ (chapter - 1)` | {1, 2, 4, 8, 16} | 装备词条乘数，决定装备强度代际差 |
| attrScale | `1.0 + (chapter - 1) × 0.15` | {1.0, 1.15, 1.30, 1.45, 1.60} | 属性点效果随章节缓慢增值，防止后期贬值 |
| scaleMul 起始值 | 上一章末关 scaleMul × 1.5 左右 | — | 确保过渡关"上一章毕业装可轻松通过" |
| scaleMul 增长 | 每关约 ×1.15~1.25 递增 | — | 装备墙关卡可到 ×1.3，过渡关低至 ×1.07 |
| 装备基础值 | `base × tierMul × (1 + quality加成)` | atk=2.0, hp=10.0, def=1.3 | 基础值较高，使装备在总属性中占比 26%~43% |
| 每点 ATK | `2 × attrScale` | ch1=2, ch5=3.2 | 降低裸点占比，强化装备重要性 |
| 暴击溢出转换 | `溢出% × 3 → 暴伤%` | 1:3 | 溢出不浪费，但仍不如直接堆暴伤（效率 60%） |

**装备门槛设计**：上一章毕业装进入下一章，玩家总战力约为本章毕业装的 **65%**，必须刷取本章装备才能推进。

### 1.3 视觉色调规则

每章主色调应与前几章形成视觉反差，避免相邻章节颜色接近。参考色环选择对比色或互补色。

## 二、章节数据结构（必填）

### 2.1 章节定义 — `StageConfig.CHAPTERS[]`

```lua
{
    id = N,                    -- [必填] 章节编号，从 1 递增
    name = "章节名称",         -- [必填] 2-4 个汉字
    desc = "一句话描述",       -- [必填] 8-15 字，显示在章节选择界面
    lore = "章节背景故事...",   -- [必填] 80-150 字，章节叙事文本
    stages = { ... },          -- [必填] 10 个关卡配置，见 §2.2
}
```

### 2.2 关卡定义 — `stages[]`

每章固定 **10 关**，结构如下：

```lua
{
    name = "关卡名称",         -- [必填] 2-5 个汉字
    isBoss = true/nil,         -- [选填] Boss 关标记，仅第 5、10 关设为 true
    waves = {                  -- [必填] 2-3 波怪物
        { monsters = {
            { id = "怪物ID", count = 数量 },
            ...
        }},
    },
    scaleMul = 数值,           -- [必填] 难度缩放系数，见 §2.3
    reward = {                 -- [必填] 通关奖励
        gold = 数值,           -- [必填] 金币奖励
        guaranteeEquipQuality = N,  -- [选填] 保底装备品质 (1=白 2=蓝 3=紫 4=橙)
    },
}
```

### 2.3 关卡分布模板（固定模式）

| 关卡 | 类型 | scaleMul 增幅 | 设计意图 | 第1章参考值 |
|------|------|---------------|----------|-------------|
| 1    | 过渡关 | 起始值（×1.07~1.10） | 上一章毕业装轻松通过，适应新怪物 | 1.4 |
| 2    | 普通关 | ×1.07~1.15 | 引入 1-2 种新怪物 | 1.5 |
| 3    | 普通关 | ×1.13~1.20 | 开始感到吃力 | 1.7 |
| 4    | **装备墙 1** | ×1.20~1.30 | 需要本章 2-3 件蓝装 | 2.1 |
| 5    | **中章 Boss** | ×1.25~1.35 | 需要 2 件套触发，isBoss=true | 2.8 |
| 6    | 普通关 | ×1.20~1.30 | Boss 后难度继续攀升 | 3.6 |
| 7    | **装备墙 2** | ×1.15~1.25 | 需要 4 件套触发或混搭 | 4.3 |
| 8    | 普通关 | ×1.20~1.25 | 全身紫装级别 | 5.3 |
| 9    | **装备墙 3** | ×1.15~1.20 | 需要橙装 + 满级强化 | 6.3 |
| 10   | **章末 Boss** | ×1.20~1.25 | 最终检定，isBoss=true | 7.8 |

> **注意**：增幅不再是均匀 ×1.2。前期过渡关增幅较小（×1.07），装备墙关卡增幅较大（×1.30），形成"缓-急"节奏感。

## 三、怪物设计（必填 8 个普通怪 + 2 个 Boss）

### 3.1 普通怪物模板 — `StageConfig.MONSTERS`

```lua
怪物ID = {
    -- === 基础属性（必填）===
    name = "显示名称",         -- 2-4 个汉字
    hp = 数值,                 -- 生命值
    atk = 数值,                -- 攻击力
    def = 数值,                -- 防御力
    speed = 数值,              -- 移动速度 (0=静止, 15~30=慢, 40~55=中, 60~80=快)
    atkInterval = 数值,        -- 攻击间隔（秒）(0.8~2.0)
    element = "元素类型",      -- fire/ice/poison/water/arcane/physical

    -- === 掉落（必填）===
    expDrop = 数值,            -- 经验掉落
    goldDrop = { 最小, 最大 }, -- 金币掉落范围

    -- === 美术（必填）===
    image = "Textures/mobs/怪物ID.png",  -- 贴图路径
    radius = 数值,             -- 碰撞半径 (12~19)
    color = { R, G, B },       -- 主题色 RGB

    -- === 元素抗性（选填，第四章起必填）===
    resist = {                 -- 正值=抗性(减伤), 负值=弱点(增伤)
        fire = 0.0,            -- 范围: -0.30 ~ 0.50
        ice = 0.0,
        poison = 0.0,
        water = 0.0,
        arcane = 0.0,
        physical = 0.0,
    },

    -- === 特殊机制（选填，按需添加）===
    antiHeal = true,                  -- 反治疗标记
    slowOnHit = 0.30,                 -- 命中减速率
    slowDuration = 2.0,               -- 减速持续秒数
    defPierce = 0.30,                 -- 无视防御比例
    isRanged = true,                  -- 远程攻击
    packBonus = 0.40,                 -- 狼群增伤比例
    packThreshold = 4,                -- 狼群触发数量
    lifesteal = 0.15,                 -- 吸血比例（攻击回血）
    firstStrikeMul = 2.0,             -- 首次攻击伤害倍率
    hpRegen = 0.03,                   -- 每 N 秒回复 HP 百分比
    hpRegenInterval = 5.0,            -- 回血间隔
    healAura = { pct=0.05, interval=8.0, radius=100 },  -- 治疗光环
    deathExplode = { element="arcane", dmgMul=1.0, radius=50 },  -- 死亡爆炸
}
```

### 3.2 Boss 模板

Boss 在普通怪基础上增加以下字段：

```lua
boss_xxx = {
    -- ... 基础属性同上 ...
    isBoss = true,             -- [必填] Boss 标记

    -- === Boss 专属技能（至少选 1-2 个）===

    -- 弹幕技能
    barrage = {
        interval = 8.0,        -- CD 秒数
        count = 8,             -- 弹幕数量
        dmgMul = 0.6,          -- 每颗伤害倍率(×ATK)
        element = "元素",
    },

    -- 减伤护甲（HP 低于阈值触发）
    iceArmor = {
        hpThreshold = 0.5,     -- HP 百分比阈值
        dmgReduce = 0.55,      -- 减伤比例
        duration = 4.0,        -- 持续秒数
        cd = 14.0,             -- 冷却秒数
    },

    -- 减速领域（HP 低于阈值触发）
    frozenField = {
        hpThreshold = 0.6,
        slowRate = 0.5,
        duration = 8.0,
        cd = 18.0,
    },

    -- 低血回血
    iceRegen = {
        hpThreshold = 0.25,    -- HP 低于此值开始回血
        regenPct = 0.02,       -- 每秒回复 HP 百分比
    },

    -- 召唤小怪
    summon = {
        interval = 12.0,       -- CD 秒数
        monsterId = "怪物ID",  -- 召唤的怪物类型
        count = 2,             -- 每次召唤数量
    },
}
```

### 3.3 怪物数值基准

怪物 HP/ATK 应参照上一章同定位怪物，按 **50-80% 上浮**：

| 定位 | HP 范围 | ATK 范围 | DEF 范围 | 速度 | 代表 |
|------|---------|----------|----------|------|------|
| 蜂群填充 | 极低(30-50) | 中低(15-22) | 0 | 高(55-70) | 墓穴鼠、灰烬鼠 |
| 高速脆皮 | 极低(20-50) | 高(25-35) | 0-2 | 高(55-75) | 暗影刺客、虚隙蝠 |
| 中等杂兵 | 中(70-100) | 中(15-25) | 4-8 | 中(40-55) | 尸蛛、亡灵侍祭 |
| 肉盾坦克 | 高(120-280) | 低(10-20) | 高(14-24) | 慢(15-30) | 骨傀儡、永冻傀儡 |
| 综合精英 | 中高(130-180) | 中高(20-28) | 中(10-15) | 中(40-50) | 诅咒骑士 |
| 中章 Boss | ×3~5 倍精英HP | ×2 倍精英ATK | 高 | 慢(20-30) | 骨冠领主 |
| 章末 Boss | ×2 倍中章Boss | ×1.3 倍中章Boss | 更高 | 慢(18-25) | 墓域君王 |

### 3.4 怪物 ID 命名规范

- 格式：`小写英文_下划线分隔`
- 普通怪：描述性名称（如 `grave_rat`、`skeleton_warrior`）
- Boss：`boss_` 前缀（如 `boss_bone_lord`、`boss_tomb_king`）
- 避免缩写，保持可读性

### 3.5 元素抗性设计原则

| 规则 | 说明 |
|------|------|
| 主题一致性 | 亡灵抗毒、冰系抗冰、火系抗火——符合直觉 |
| 至少一个弱点 | 每种怪至少有一个 -20% 以上的弱点元素，避免无解 |
| 综合精英无极端 | 精英怪不设 >40% 的极端抗性，保持多元素可打 |
| 毒/物理分化 | 可设定某章整体高毒抗，迫使玩家调整元素策略 |
| Boss 不重复弱点 | 中章 Boss 和章末 Boss 弱点应不同，避免一套打穿 |

抗性数值含义：

| 数值 | 含义 | 玩家感知 |
|------|------|----------|
| -30% | 大弱点 | 伤害提升 30%，最优选择 |
| -20% | 弱点 | 推荐使用该元素 |
| 0% | 中性 | 正常伤害 |
| 10~20% | 轻微抗性 | 打得动但不推荐 |
| 30~40% | 高抗 | 明显肉，换元素 |
| 50% | 免疫级 | 伤害减半，基本打不动 |

## 四、套装设计（必填 2 套）

### 4.1 设计原则

每章 **一攻一守**，两套套装，每套 **2/4/6 件** 三档套装效果：

| 类型 | 定位 | 2 件效果 | 4 件效果 | 6 件效果 |
|------|------|----------|----------|----------|
| 攻击套 | 输出向 | 增伤/增速类被动 | 主动触发型爆发（需积累条件） | 终极爆发/多机制联动 |
| 防御套 | 生存向 | 抗性/防御类被动 | 免死/回血/反击类保命技 | 极限生存/反击风暴 |

### 4.2 套装数据结构 — `Config.EQUIP_SETS[]`

```lua
{
    id = "套装ID",             -- [必填] 小写英文下划线，全局唯一
    name = "套装名称",         -- [必填] 2-4 个汉字
    chapter = N,               -- [必填] 所属章节
    color = { R, G, B },       -- [必填] 套装主题色

    bonuses = {
        -- 2 件效果
        [2] = {
            desc = "效果描述文本",      -- [必填] 显示用
            stats = { ... },            -- [选填] 直接加属性
            statsMul = { ... },         -- [选填] 百分比加属性
            resist = { ... },           -- [选填] 加抗性
            buff = { ... },             -- [选填] 触发型效果
        },
        -- 4 件效果
        [4] = {
            desc = "效果描述文本",
            buff = { ... },             -- 4 件通常是复杂触发效果
        },
        -- 6 件效果（终极效果）
        [6] = {
            desc = "效果描述文本",
            buff = { ... },             -- 6 件通常是最强触发效果，联动前序效果
        },
    },
}
```

### 4.4 套装与怪物机制的联动

**核心设计原则**：套装效果应直接克制本章怪物的核心机制。

示例（第四章）：
- 诅咒骑士有"吸血" → 净魂烈焰 4 件提供"反治疗 -70%"
- 大量奥术怪 → 怨魂之怒 2 件提供"奥术抗性 +30%"
- 亡灵侍祭有"治疗光环" → 净魂烈焰 AOE 爆炸可快速清场

### 4.5 装备需求曲线

每章应有清晰的装备驱动路径。由于 **装备门槛约 65%**（上一章毕业装 vs 本章毕业装的战力比），玩家必须在本章刷取装备才能推进：

| 阶段 | 推荐配置 | 战力水位 | 说明 |
|------|----------|---------|------|
| 第 1-3 关 | 上一章旧装 | ~65% | 无需换装即可通过，旧装仍有余力 |
| 第 4 关（墙 1） | 本章 2 件套 | ~72% | 2 件套效果带来质变，突破第一道墙 |
| 第 5 关（Boss） | 至少 1 套 2 件触发 | ~78% | Boss 机制需要套装对策 |
| 第 6-7 关 | 2+2 混搭 或开始凑 4 件 | ~83% | 攻守兼备 |
| 第 8 关（墙 2） | 任一 4 件套 | ~88% | 4 件效果是质变 |
| 第 9 关（墙 3） | 4+2 混搭 或开始凑 6 件 | ~94% | 向6件套冲刺 |
| 第 10 关（Boss） | 完整 6 件套 + 高强化 | ~100% | 终极检定，毕业装水平 |

> **设计原则**：装备 Tier 系数 `2^(ch-1)` 使相邻章节装备差距 ×2。配合 scaleMul 调整，玩家每获得一件本章装备约提升 5~8% 战力，形成持续正反馈。

## 五、美术资源清单（必填）

### 5.1 需要的图片

每章需要准备以下图片资源：

| 类型 | 数量 | 尺寸 | 比例 | 透明背景 | 说明 |
|------|------|------|------|----------|------|
| 关卡选择背景 | 1 张 | 512×1024 | 4:5 | 否 | 竖屏全屏背景，展示章节主题场景 |
| 战斗背景 | 1 张 | 1024×512 | 16:9 | 否 | 横屏战斗区域背景，2D 俯视角 |

## 六、关卡怪物搭配原则

### 6.1 难度曲线

| 阶段 | 怪物搭配策略 | 目的 |
|------|-------------|------|
| 1-2 关 | 1-2 种简单怪，纯数量 | 让玩家适应新怪物 |
| 3 关 | 引入高威胁怪（穿透/爆发） | 考验生存能力 |
| 4 关（墙） | 核心机制组合（肉盾+治疗+吸血等） | 迫使换装 |
| 5 关 Boss | Boss + 大量杂兵 | 第一波 清杂兵，第二波 Boss 登场 |
| 6-7 关 | 全种类混合 | 综合考验 |
| 8-9 关（墙） | 最难怪物大量堆叠 | 终极装备检定 |
| 10 关 Boss | Boss + 精锐随从 | 章节最终检定 |

### 6.2 Boss 关固定结构

Boss 关必须至少 2 波：
- **第 1 波**：大量杂兵（清场热身）
- **第 2 波**：Boss × 1 + 少量杂兵（核心战斗）

### 6.3 每波怪物数量参考

| 怪物类型 | 单波数量范围 |
|----------|-------------|
| 蜂群填充 | 15 ~ 35 |
| 高速脆皮 | 8 ~ 15 |
| 中等杂兵 | 6 ~ 12 |
| 肉盾坦克 | 3 ~ 8 |
| 综合精英 | 5 ~ 12 |
| Boss | 1 |

## 九、数值体系参考

> 新增章节时，必须参照以下参数确保数值一致性。

### 9.1 属性点系统

| 属性 | 每点效果 | 特殊机制 |
|------|---------|---------|
| ATK | +2 ATK × attrScale | ch1 实际+2，ch5 实际+3.2 |
| SPD | +0.05 攻速 | 下限 0.1 |
| CRT | +2% 暴击率 | 上限 100%，溢出按 1:3 转暴伤 |
| CDM | +10% 暴击伤害 | 无上限 |
| RNG | +8 攻击范围 | 无上限 |
| LCK | +3% 幸运 | 影响掉落品质权重和金币量 |
| VIT | 复合（见下方） | hp/def 受 attrScale 缩放 |
| TEN | +2% 通用减益抗性 | 上限 80%，作用于减速/减疗/腐蚀/致盲 |

### 9.2 VIT 每点详细效果

| 子属性 | 每点效果 | 受 attrScale 影响 | 说明 |
|--------|---------|-------------------|------|
| HP | +15 | 是 | ch1=+15, ch5=+24 |
| DEF | +0.3 | 是 | ch1=+0.3, ch5=+0.48 |
| HP Regen | +1.0/秒 | 否 | 固定值 |
| Heal% | +0.5% | 否 | 治疗倍率 |
| Shield% | +0.3% | 否 | 护盾倍率 |
| hpPct | +0.2% MaxHP | 否（乘算） | 后期指数增长，335 VIT → +67% MaxHP |

### 9.3 韧性 (TEN) 通用减益抗性

韧性作用于**所有** debuff，而非仅减速：

| debuff 类型 | 效果衰减 | 持续时间衰减 |
|-------------|---------|-------------|
| 减速 | `slowRate × (1 - resist)` | `dur × (1 - resist × 0.5)` |
| 减疗 | `rate × (1 - resist)` | `dur × (1 - resist × 0.5)` |
| 腐蚀(降DEF) | `defReduce × (1 - resist)` | `dur × (1 - resist × 0.5)` |
| 墨汁致盲(降ATK) | `atkReduce × (1 - resist)` | `dur × (1 - resist × 0.5)` |

设计新怪物 debuff 时，应考虑韧性的衰减效果。20 点韧性（40%抗性）会将 debuff 效果削弱近半。

### 9.4 装备词条基础值

| 词条 | base 值 | 说明 |
|------|---------|------|
| atk | 2.0 | 主要输出词条 |
| hp | 10.0 | 主要生存词条 |
| def | 1.3 | 防御词条 |
| hpRegen | 0.67 | 回血词条 |
| lifeSteal | 0.08% | 吸血词条 |
| shldPct | 0.1% | 护盾比例 |
| 元素抗性 | 0.8% | fireRes/iceRes 等 |

实际装备词条值 = `base × tierMul × (1 + 品质加成) × 随机波动`

### 9.5 新章节 scaleMul 计算方法

新增第 N 章时，按以下步骤计算 scaleMul：

1. **确定起始值**：上一章第 10 关 scaleMul × 1.3~1.5（玩家 DPS 提升空间）
2. **确定终止值**：起始值 × 5~6（章内增长约 5~6 倍）
3. **分配 10 关**：按 §2.3 的增幅模板分配，装备墙关卡给更高增幅
4. **验证**：确认第 1 关上一章毕业装可通过，第 4 关需要本章装备

**参考各章增长倍率**（末关 / 首关）：

| 章节 | 首关 | 末关 | 章内倍率 |
|------|------|------|---------|
| 1 | 1.4 | 7.8 | ×5.6 |
| 2 | 12 | 60 | ×5.0 |
| 3 | 88 | 422 | ×4.8 |
| 4 | 600 | 3,460 | ×5.8 |
| 5 | 4,600 | 24,600 | ×5.3 |
