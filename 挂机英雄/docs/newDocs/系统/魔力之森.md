## §1 系统概览

### 1.1 设计定位

魔力之森是一个以 **魔力（Mana）** 为核心机制的限时挑战副本。玩家进入一片充满魔力能量的远古森林，森林中的怪物吸收了过量魔力发生异变。击杀怪物会释放 **魔力精华**，玩家可以在战斗中收集精华来获得临时增益，战斗结束后精华转化为 **魔力药水** 和 **森之露**（魔力之源升级专属材料）。

**核心体验**: 在有限时间内尽可能多地收集魔力精华，利用精华带来的增益形成正反馈循环——越多精华 → 越强增益 → 越快击杀 → 越多精华。

### 1.2 模式定位对比

| 副本 | 核心目的 | 限时 | 每日次数 | 独特机制 |
|------|---------|------|---------|---------|
| 无尽试炼 | 经验 / 推进 | 否 | 无限 | 无限层数推进 |
| 深渊模式 | 深渊之心 / 怨魂碎片 | 否 | 无限 | 动态等级、无限层 |
| 套装秘境 | 定向套装掉落 | 120s | 3次 | 选择目标套装 |
| 世界Boss | 排行榜 / 竞争 | 60s | 3次 | 伤害排名 |
| 资源秘境 | 宝石 / 棱镜 | 60s | 3次 | 批量宝石奖励 |
| **魔力之森** | **魔力药水 / 森之露** | **90s** | **3次** | **精华收集 → 增益循环** |

### 1.3 模块结构

| 模块 | 文件 | 职责 |
|------|------|------|
| 核心逻辑 | `scripts/ManaForest.lua` | 副本状态、怪物生成、精华系统、结算 |
| 战斗适配 | `scripts/ManaForest.lua` (GameMode adapter) | 接入 GameMode 框架 |
| 入口 UI | 复用现有副本入口面板 | 次数显示、难度选择、进入按钮 |
| 结算 UI | 复用 RewardPanel | 精华统计、奖励展示 |

## §2 进入条件与次数

### 2.1 解锁条件

| 条件 | 值 | 说明 |
|------|---|------|
| 章节要求 | 第 4 章通关 | 此时玩家已接触魔力药水系统 |
| 困难模式解锁 | 第 8 章通关 | 怪物更强，奖励更好 |

### 2.2 每日次数

| 参数 | 值 | 说明 |
|------|---|------|
| 每日免费次数 | 3 | 与其他副本一致 |
| 额外次数 | 看广告 +1 次（每日上限 2 次） | 广告变现入口 |
| 次数重置 | 每日 00:00 | 跟随系统日期重置 |

### 2.3 失败处理

- 超时未击杀任何怪物 → 不消耗次数
- 超时但已有收集 → 消耗次数，按已收集精华结算
- 玩家死亡 → 消耗次数，按已收集精华结算（60% 效率）

## §3 战斗机制

### 3.1 基础参数

| 参数 | 普通模式 | 困难模式 |
|------|---------|---------|
| 战斗时长 | 90 秒 | 90 秒 |
| 怪物缩放 | `maxChapterScale × 0.80` | `maxChapterScale × 1.30` |
| 怪物总数 | 30 | 40 |
| 精英数量 | 3 | 5 |
| 精英HP倍率 | ×2.5 | ×3.5 |
| Boss | 无 | 无 |

### 3.2 怪物生成

采用**持续涌入**模式，而非波次制：

```
场上同时存在怪物数上限 = 8（普通）/ 10（困难）
每当场上怪物少于上限 → 立即补充
生成间隔 = 0.8 秒
精英在第 10 / 20 / 30 只（普通）或第 8 / 16 / 24 / 32 / 40 只（困难）出现
```

**怪物家族**: 使用森林主题怪物（通过 MonsterFamilies 系统）：
- 普通怪：使用 `swarm` 行为模板（低血量、高数量）
- 精英怪：使用 `bruiser` 或 `tank` 行为模板（高血量、高奖励）

### 3.3 魔力精华系统（核心机制）

击杀怪物后，除了正常掉落（金币、经验）外，额外掉落 **魔力精华球**：

| 怪物类型 | 精华掉落量 |
|---------|-----------|
| 普通怪 | 1 |
| 精英怪 | 5 |

精华球的行为与金币类似（散落 → 延迟 → 自动吸收），但使用独特的蓝绿色视觉效果。

**精华收集 → 增益阶梯**：

| 累计精华 | 增益效果 | 视觉表现 |
|---------|---------|---------|
| 0-4 | 无增益 | 无 |
| 5-14 | 攻速 +15% | 角色淡蓝光环 |
| 15-24 | 攻速 +15%，伤害 +10% | 蓝色光环增强 |
| 25-34 | 攻速 +30%，伤害 +20% | 蓝紫色光环 |
| 35+ | 攻速 +30%，伤害 +20%，暴击率 +10% | 紫色炫光环绕 |

**设计意图**：
- 增益不会过于夸张（最大档约等于一件紫装的加成），避免让副本内战力膨胀失控
- 阶梯式设计让玩家有持续收集的动力
- 增益仅在本次副本内生效，退出后清除

### 3.4 魔力涌潮（随机事件）

战斗过程中，每 30 秒触发一次 **魔力涌潮**：

1. 屏幕提示"魔力涌潮！"（1.5 秒预警）
2. 随机在场地中央区域生成 **3 个魔力水晶**
3. 水晶存在 8 秒后消失
4. 每个水晶被玩家角色碰触后获得 3 精华

**AI 自动拾取**: 由于游戏为挂机模式，魔力水晶需要注册为 AI 可识别的拾取目标。当场上存在水晶时，AI 将水晶视为**高优先级目标**（优先级高于普通怪、低于精英怪），主动移动到水晶位置拾取：

```lua
-- 在 AI 目标选择逻辑中注册水晶为可交互目标
-- 优先级: 精英怪 > 魔力水晶 > 普通怪
AITarget.Register("mana_crystal", {
    priority  = 75,       -- 精英=100, 普通怪=50
    range     = 999,      -- 全场感知
    action    = "move_to", -- 移动到位置即触发拾取
    onReach   = function(crystal)
        ManaForest.CollectCrystal(crystal)
    end,
})
```

**设计意图**: 涌潮水晶作为额外精华来源，AI 会自动处理拾取，玩家无需手动操作但可以观察到角色为拾取水晶而改变移动路线的有趣行为。

## §4 奖励设计

### 4.1 新材料：森之露

**森之露** 是魔力之森副本的**独占掉落材料**，用于升级魔力之源。不在任何其他系统中产出，保证副本的唯一价值。

```lua
-- 在 Config.MATERIAL_DEFS 中追加
{ id = "forestDew", name = "森之露", rarity = "rare", maxStack = 999,
  desc = "魔力之森中凝结的纯净魔力，可用于强化魔力之源", icon = "image/forest_dew.png" }
```

### 4.2 精华转化（核心奖励）

战斗结束后，收集的精华转化为实际奖励：

| 奖励类型 | 普通模式 | 困难模式 | 说明 |
|---------|---------|---------|------|
| 魔力药水 | `floor(精华 / 8)` 瓶 | `floor(精华 / 6)` 瓶 | 补充消耗品 |
| 森之露 | `floor(精华 / 7)` | `floor(精华 / 5)` | **核心产出** |

**典型收益估算**（假设 90 秒内收集约 35-45 精华）：

| | 普通模式（~35精华） | 困难模式（~45精华） |
|---|---|---|
| 魔力药水 | 4 瓶 | 7 瓶 |
| 森之露 | **5** | **9** |

### 4.3 额外奖励（固定）

每次通关额外获得：

| 奖励 | 普通模式 | 困难模式 |
|------|---------|---------|
| 金币 | `200 × scaleMul^0.3` | `400 × scaleMul^0.3` |
| 经验 | `2000 × scaleMul` | `4000 × scaleMul` |

> `scaleMul^0.3` 即 `ConfigCalc.GetGoldScale(scaleMul)`，金币增速低于怪物强度增速，与主线保持一致。

### 4.4 首通奖励

每日首次通关额外获得：

| 奖励 | 说明 |
|------|------|
| 魔力药水 ×5 | 固定奖励 |
| 森之露 ×3 | 固定奖励 |

### 4.5 魔力之源升级成本表

魔力之源共 **10 级**，每级效果 +3%（30% → 60%）。升级材料从广告改为**森之露**：

| 等级 | 森之露消耗 | 累计消耗 | 预计天数（4 次/天） | 累计天数 | 效果 |
|------|----------|---------|-------------------|---------|------|
| 1 | 20 | 20 | 1 天 | 1 | 33% |
| 2 | 35 | 55 | 1.8 天 | ~3 | 36% |
| 3 | 55 | 110 | 2.8 天 | ~6 | 39% |
| 4 | 80 | 190 | 4 天 | ~10 | 42% |
| 5 | 110 | 300 | 5.5 天 | ~15 | 45% |
| 6 | 145 | 445 | 7.3 天 | ~23 | 48% |
| 7 | 185 | 630 | 9.3 天 | ~32 | 51% |
| 8 | 230 | 860 | 11.5 天 | ~43 | 54% |
| 9 | 280 | 1140 | 14 天 | ~57 | 57% |
| 10 | 340 | 1480 | 17 天 | ~74 | 60% |

> **校准依据**：
> - 每日 4 次（3 免费 + 1 广告），普通模式约 5 森之露/次 = **~20 森之露/天**
> - 等级 1 消耗 20 → 第 1 天 4 次即可升 1 级（符合用户要求）
> - 总计 1480 森之露 ÷ 20/天 = **74 天 ≈ 2.5 个月**（纯普通模式）
> - 解锁困难模式后（Ch8），产出提升至 ~28/天 → 后期加速，总计约 **3 个月**到满级
> - 前 5 级（~15 天）给予明显体感提升（30% → 45%），后 5 级为长线目标

**升级 UI 说明**：在魔力之源面板中，原广告升级按钮改为"消耗 X 森之露 → 升级"按钮，材料不足时置灰并显示 `拥有/需要`。

### 4.6 经济性分析

| 获取途径 | 魔力药水/次 | 森之露/次 | 定位 |
|---------|-----------|----------|------|
| 魔力之森（普通） | 4 | **5** | 药水 + 升级材料 |
| 魔力之森（困难） | 7 | **9** | 药水 + 升级材料（高效） |
| 看广告 | 1 小时无消耗回复 | 0 | 仅回复，不产出材料 |

**结论**：
- **森之露是魔力之森的独占产出**，不与任何现有材料经济冲突
- 魔力之森是魔力药水的稳定获取途径 + 魔力之源的唯一升级途径
- 副本具备长线价值（3 个月升满），不会因满级而失去意义（药水仍然需要）

## §6 GameState 数据结构

```lua
GameState.manaForest = {
    attemptsToday = 0,          -- 今日已用次数
    bonusAttempts = 0,          -- 广告额外次数（已用）
    lastDate      = "",         -- 上次重置日期
    totalRuns     = 0,          -- 历史总通关次数
    bestEssence   = 0,          -- 单次最高精华记录
    totalEssence  = 0,          -- 历史总精华收集
    firstClearToday = false,    -- 今日是否已首通
}
```

### 6.1 每日重置逻辑

```lua
function ManaForest.EnsureState()
    local today = os.date("%Y%m%d")
    if GameState.manaForest.lastDate ~= today then
        GameState.manaForest.attemptsToday = 0
        GameState.manaForest.bonusAttempts = 0
        GameState.manaForest.firstClearToday = false
        GameState.manaForest.lastDate = today
    end
end
```

## §8 GameMode Adapter 接口

```lua
local adapter = {}
adapter.background = "image/battle_bg_forest.png"  -- 森林背景

function adapter:OnEnter()
    return ManaForest.EnterFight()
end

function adapter:OnExit()
    ManaForest.ExitFight()
end

function adapter:BuildSpawnQueue()
    return ManaForest.BuildSpawnQueue()
end

function adapter:GetBattleConfig()
    return {
        isBossWave = false,
        bossTimerMax = 90,                -- 90 秒限时
        startTimerImmediately = true,
    }
end

function adapter:IsTimerMode()
    return true
end

function adapter:OnEnemyKilled(bs, enemy)
    ManaForest.OnEnemyKilled(bs, enemy)
    return false  -- 允许正常掉落
end

function adapter:OnTimeout(bs)
    ManaForest.EndFight(true)  -- 超时 = 正常结算
    return true
end

function adapter:OnDeath(bs)
    ManaForest.EndFight(false) -- 死亡 = 60% 效率结算
    return true
end

function adapter:CheckWaveComplete(bs)
    -- 持续涌入模式：不通过波次判定结束
    -- 仅通过超时 / 击杀全部 / 死亡结束
    if ManaForest.AllSpawned() and ManaForest.AllKilled() then
        ManaForest.EndFight(true)
        return true
    end
    return false
end

function adapter:DrawWaveInfo(nvg, l, bs, alpha)
    ManaForest.DrawHUD(nvg, l, bs, alpha)
end

GameMode.Register("manaForest", adapter)
```

| 增益数值 | 攻速15-30%，伤害10-20%，暴击10% | ±5% | 不应让玩家一刀秒 |
| 困难模式缩放 | ×1.30 | 1.10-1.50 | 确保有挑战但可通关 |
| 涌潮间隔 | 30s | 20-40s | 90s 内触发 2-3 次为宜 |
| 涌潮水晶精华 | 3/个 | 2-5 | 水晶收集应有可感知收益 |
| 场上怪物上限 | 8/10 | 6-12 | 太多影响性能，太少节奏慢 |
| 广告额外次数上限 | 2 | 1-3 | 广告变现与体验的平衡 |

## §15 设计理念

### 为什么需要魔力之森？

1. **魔力之源升级途径缺失**: 当前魔力之源只能通过看广告升级，没有"玩出来"的获取途径。森之露作为独占材料，让升级变成可刷取的长线目标。

2. **魔力药水的稳定获取**: 药水同时作为副本产出，补充日常消耗。

3. **正反馈循环**: 精华 → 增益 → 更快击杀 → 更多精华，让玩家在 90 秒内体验到明显的战力增长感。

4. **与魔力系统联动**: 作为"魔力之源"药水系统的自然延伸，主题和机制高度统一。副本产出（森之露）直接驱动魔力之源成长，形成"刷副本 → 升级药水 → 战斗更强 → 刷更难副本"的闭环。

5. **长线留存价值**: 3 个月的升级周期确保副本在长期内保持价值，满级后仍需刷药水维持消耗。

### 为什么没有 Boss？

- 90 秒限时 + 持续涌入的设计强调"效率"而非"硬扛"
- Boss 会打断精华收集节奏
- 精英怪已提供足够的战斗压力
- 与深渊模式（有 Boss）和世界 Boss（纯 Boss）差异化
